# Track Log Studio 使用手冊

> 本手冊隨功能更新持續擴充，若畫面與描述略有出入，請以實際軟體介面為準。

---

## 1. 這是什麼

**Track Log Studio** 是一個**純瀏覽器端**的賽道記錄工具，不需要安裝、不需要帳號，也**不會把你的任何檔案上傳到伺服器**——所有解析、轉檔、分析都在你的瀏覽器裡完成。

兩大功能：

1. **轉換（Converter）**：把 aRacer ECU 的 `.loga` 記錄檔轉換成 [RaceChrono](https://racechrono.com/) DIY 可匯入的 `.nmea`（NMEA 0183 + `$RC3` 感測器槽位）、Racelogic `.vbo`（給 Circuit Tools 3 / RaceChrono 使用），或通用 `.csv`（給 Race Studio 3 / Excel / Python 等任何工具使用）。
2. **分析（Analyzer）**：直接在瀏覽器裡看軌跡地圖、切圈、比較圈速、畫遙測圖表、看賽道熱力圖。

> **商標聲明：** Track Log Studio 為獨立的非官方工具，與 aRacer、RaceChrono 無任何隸屬或背書關係；文中提及僅用於描述相容性。aRacer 與 RaceChrono 為其各自所有者的商標。

## 2. 開始使用

### 2.1 開啟網站

直接用瀏覽器開啟網站即可使用，無需註冊登入。建議使用近期版本的 Chrome / Edge / Safari（手機、平板、桌面皆可，畫面會自動依螢幕大小調整，也可手動調整版面）。

**手機導覽列：** 螢幕寬度較窄（約手機尺寸，768px 以下）時，「轉換／分析／設定」會改用畫面底部的分頁列（貼齊 iPhone Home Indicator 安全區域），取代桌面版上方的分頁；桌面／平板寬度則維持上方分頁列。切換分頁時畫面會有輕微滑動＋淡入淡出效果，若系統開啟「減少動態效果」（prefers-reduced-motion），會自動改為純淡入淡出。

### 2.2 安裝成 App（PWA）

本站是一個 **PWA（Progressive Web App）**，可以「安裝」到手機或電腦桌面，像原生 App 一樣使用：

- **手機（iOS / Android）**：瀏覽器選單中選擇「加入主畫面」或「安裝應用程式」。
- **桌面（Chrome / Edge）**：網址列右側會出現安裝圖示，點擊即可安裝。

安裝後可**離線使用**（已載入過的頁面與程式碼會被快取），但每次轉檔、分析仍需要你自己上傳/選取記錄檔——記錄檔本身不會被預先快取或上傳雲端。

### 2.3 支援的記錄來源

**經測試支援：**

| 來源 | 說明 |
|---|---|
| RC super2 | 透過 SpeedTuning 2 回讀 |
| RC superX | 透過 SpeedTuningX 回讀 |
| aRacer X tune App | 透過分享功能輸出 log（可直接上傳 `.zip`，會自動解壓） |
| aRacer Logger 2.5 Module | 透過 Logger2 Reader 讀出 |

**理論上支援但尚未測試：** RC super、RC superXX、RC mini X、RC mini XX、aRacer Race Module 3

### 2.4 可匯入的檔案格式

| 副檔名 | 說明 |
|---|---|
| `.loga` | aRacer ECU 原始記錄檔（Super2 / SuperX / RaceAMP / aRacer X tune App 等檔頭皆自動相容） |
| `.nmea` | 先前已轉出的 NMEA/RC3 記錄 |
| `.vbo` | Racelogic VBOX 格式（含本站匯出的、或其他來源的 VBO） |
| `.rcz` | RaceChrono 記錄檔（ZIP 內含 session.json + 二進位通道資料） |
| `.xrk` | AiM Solo 2 DL / MyChron5 記錄檔 |
| `.rcnx` | Qstarz LT-Q6000 / Q6000S（QRacing）記錄檔 |
| `.zip` | aRacer x Tune App 分享輸出的壓縮檔，上傳後會自動解壓並辨識內容 |

上傳時系統會依副檔名與檔案內容自動判斷格式，不需要自己選擇「匯入哪種格式」。

#### RCNX 多 session

一個 `.rcnx`檔案可能包含多個 session（多次記錄）。若偵測到超過一個 session，載入時會跳出「選擇要開啟的一個」的挑選視窗，列出每個 session 的點數、時長（分鐘）、是否含圈速資料，並把資料量最多的一個標示為「（資料最多）」建議選項。點擊任一 session 即可開啟；「取消」則放棄這次匯入。若該 session 對應的分析資料（`sana` 資料庫）中已有圈速紀錄，圈次會自動以此為來源匯入（即 ECU 切圈，不需要再手動拖曳起終點線）。

### 2.5 可匯出的檔案格式

- RaceChrono DIY `.nmea`（NMEA 0183 `$GPRMC` + `$RC3`）
- Racelogic `.vbo`（`_ct.vbo` + `_rc.vbo` + `_channels.csv` 對照表）
- 通用 `.csv`（給 Race Studio 3 / Excel / Python 等工具使用；`Time`/`GPS_Lat`/`GPS_Lon`/`GPS_Speed` + 所有其他通道，每筆取樣一列）
- 另存修改後的 `.loga`（把避震行程等衍生通道寫回新的 `.loga`；僅原始來源本身就是 `.loga` 時可選）

## 3. 轉換 Converter

轉換負責把 `.loga` 轉成 RaceChrono 可讀的 `.nmea`、Circuit Tools 3 可讀的 `.vbo`，或通用的 `.csv`。

### 3.1 載入檔案

1. 進入「轉換」頁籤。
2. 點擊「選擇檔案」或直接把檔案拖放到上傳區域，可一次選取多個 `.loga` 檔。
3. 已載入的檔案會列在「已載入檔案」清單中，顯示列數或解析中/錯誤狀態；可個別「移除」或「全部清除」。

### 3.2 選擇輸出格式

在「輸出格式」處選擇：

- **NMEA / RC3**：輸出 GPS（`$GPRMC`）+ 最多數個感測器槽位（`$RC3`），供 RaceChrono DIY 裝置匯入。需要在下方手動設定「欄位對應」。
  - 若記錄檔沒有 GPS（未安裝 aRacer Race Module），仍可轉出感測器數據，但時間戳會以轉檔當下的時間合成。
- **VBO**：自動輸出**所有** ECU 通道，不需要手動設定欄位對應。每個 `.loga` 會產生三個檔案：
  - `_ct.vbo`（Circuit Tools 用，保留原始 ECU 通道名稱）
  - `_rc.vbo`（RaceChrono 用，通道以 RaceChrono 識別符命名）
  - `_channels.csv`（通道對照表：ECU 通道 / 說明 / RaceChrono 識別符 / 單位 / 類型）
- **CSV**：自動輸出**所有**通道（含避震衍生通道），不需要手動設定欄位對應，也沒有 Race Studio 3 以外的目標軟體限制。每個 `.loga` 產生一個 `.csv`：欄位依序為 `Time`、`GPS_Lat`、`GPS_Lon`、`GPS_Speed`，之後接每個其他通道（以 aRacer 原始通道名稱命名），每筆取樣一列。無 GPS 定位的列，經緯度欄位留空；無值的取樣格留空（不會填 0）。換行採 `\n`（LF only），UTF-8 不含 BOM。
- **另存修改 (.loga)**：把校正後的避震行程等衍生通道寫回一份新的 `.loga`（不覆寫原始檔案）。**僅當目前載入的原始來源本身就是 `.loga` 時才可選**；若來源是其他格式（`.nmea`/`.vbo`/`.rcz`/`.xrk`/`.rcnx`），此選項會顯示為停用並附上提示文字。詳見 3.6 節。

### 3.3 RC3 欄位對應（僅 NMEA 模式）

NMEA/RC3 每筆最多只能塞入固定數量的感測器槽位，所以需要手動指定「哪個 loga 欄位放進哪一槽」：

- **固定欄位（自動填入）**：
  - 加速度 `xacc/yacc/zacc` ← `TC_Xforce/Yforce/Zforce` ÷ 1000（需要 Race Module IMU，沒有的話留空）
  - 角速度 `gyrox/y/z` ← `TC_Xangle_dps/Yangle_dps/Zangle_dps`（同上，需 IMU）
  - `rpm/d1` ← `RPM`
- **可自訂槽位**：逐一為每個槽位選擇要對應的 loga 欄位，也可以搜尋欄位名稱快速找到、或選「不使用」留空。

#### Preset（欄位組合）

為了不用每次都重新設定，欄位對應可以存成「Preset」：

- 目前組合下拉選單可切換：**預設**、**自訂（未儲存）**、**使用者 1–5**。
- 調整過對應後選單會顯示「自訂（未儲存）」，此時可以「儲存到」使用者 1–5 其中一組並命名。
- 「重設為預設」可以隨時把目前組合還原成系統預設值。

### 3.4 避震校正（Suspension）

若記錄含 `SuspensionAD` 電壓通道，可以在轉換頁下方可展開的「避震校正」區塊把電壓換算成行程（mm），作為衍生通道供轉檔與分析使用（詳見第 3.4.1 節）。避震設定為全站共用（分析頁引用同一份設定），只是操作介面放在轉換頁。

#### 3.4.1 避震校正參數

把 `SuspensionAD` 電壓訊號換算成行程（mm），作為衍生通道供轉檔與分析使用：

- 分別對「前避震」「後避震」設定：
  - **啟用**：是否套用此軸的校正。
  - **AD 來源**：對應到哪個 loga 電壓通道。
  - **最小電壓 / 最大電壓 (mv)**、**零點電壓 (mv)**：電壓端點與零位。
  - **最小行程 / 最大行程 (mm)**：對應到的實際行程端點。
  - **預覽**：即時顯示目前參數換算出的行程曲線／數值，方便確認校正是否合理。
  - **輸出通道**：顯示校正後會產生的通道名稱。
- **從載入檔倒算**：若手邊的記錄同時含有 `SuspensionAD` 電壓與 ECU 已知的行程數據，可以用這個功能反推校正參數（r² 為擬合優度；由於 5 個參數無法從資料唯一還原，結果是在假設電壓範圍 0–5000mv 下的其中一組解）。若資料不足、未設定或行程為定值，會提示無法倒算。

### 3.5 批次轉換與下載

1. 設定好輸出格式（與 RC3 對應，如適用）後，點擊「轉換」。
2. 轉換完成後，「轉換結果」區塊會列出每個檔案，可個別「下載」，或「全部下載 (ZIP)」一次打包下載所有結果。

### 3.6 另存修改後的 .loga

若目前載入的原始記錄本身就是 `.loga`，可以在「輸出格式」選擇「另存修改 (.loga)」；若同時在 3.4 節啟用了避震校正，且記錄檔含對應的 `SuspensionAD` 通道，就會把校正後的行程資料寫進一份**新的** `.loga`（不會覆寫原始檔案；若原本已有對應欄位會替換，沒有則新增）。可單檔「另存」，或「全部另存 (ZIP)」批次輸出。若原始來源不是 `.loga`（例如 `.nmea`/`.vbo`/`.rcz`/`.xrk`/`.rcnx`），此格式選項會停用。

## 4. 分析 Analyzer

分析讓你在瀏覽器裡直接檢視軌跡、切圈、比較遙測數據，不需要另外安裝分析軟體。

### 4.1 載入記錄

進入「分析」頁籤，點選上方「載入記錄」選取 `.loga`、`.nmea`、`.vbo`、`.rcz`、`.xrk`、`.rcnx` 或 `.zip` 檔案（可參考第 2.4 節完整格式清單）。尚未載入任何記錄時，畫面會提示你先載入檔案。

### 4.2 軌跡地圖 Track Map

- **縮放 / 平移**：滑鼠滾輪縮放、拖曳平移；觸控裝置可雙指縮放（pinch）與拖曳平移。
- **重設視角**：一鍵恢復預設縮放與位置（也可雙擊地圖快速重設）。
- **起終點線**：拖曳軌跡圖上的兩個端點即可設定起終點線位置，用來切圈；「重設起終點線」可還原預設位置。
- **軌跡通道標記**：見 4.6 節（軌跡上色與極值標記已整合為單一「選通道」控制）。

### 4.3 圈次表 Lap Table

系統會依起終點線（或 ECU 內建的切圈資訊，二擇一來源：「線段自算」/「ECU」）自動切出每一圈，列在圈次表中：

- 預設欄位：**#**（圈序）、**圈時**、**距離**。
- **新增欄位**：可加入任意通道的**最大 / 最小 / 平均**值欄位（例如「本圈最高車速」），也可以「移除欄位」。
- **新增 sector 時間**：需先設有 sector 閘門（見 4.5 節）；新增後可從下拉選單挑選「Sector 1」「Sector 2」……顯示該圈通過該段所花的時間。
- **新增差距**：加入一欄「差距」，顯示該圈圈時相對於（目前列表中未排除的）最快圈的差距，正值代表比最快圈慢。
- **標記**：
  - ⚡ 最快圈、🐢 最慢圈 會自動標示（排除掉的圈不計入評選）。
  - 每圈可手動「排除此圈」（適用切西瓜、進站、起跑等爛圈，排除後不列入最快圈評選），也可「取消排除」還原。
  - 若設有 sector 閘門，未通過 sector 檢查的圈會額外顯示提示（見 4.5 節）。
- **點選圈次**：可在表格中勾選圈次，用於下方圖表的疊圈比較模式。
- **清單為空時**：表示尚未偵測到圈次，請先拖曳起終點線或切換切圈來源。

#### 有效圈速區間過濾

可設定「有效圈速區間（秒）」的最小 / 最大值，超出此區間的圈會自動視為排除（適用於一鍵濾掉明顯過短/過長的雜訊圈，例如進出維修站的那一圈）。可隨時「清除區間」還原。此排除與手動排除的圈是聯集關係——只要符合任一條件就會被排除。

### 4.4 圖表 Charts

- **新增圖表**：點擊「新增圖表」建立一個新的時間序列圖表卡片，每張圖表可各自選擇要繪製的通道；也可以改點「新增 XY 散佈圖」建立散佈圖卡片（見下方「XY 散佈圖」）。兩種圖表可同時存在多張、各自獨立設定，並各自有「移除」按鈕。
- **X 軸模式**：可切換以「時間」或「距離」為橫軸。
- **檢視模式**：
  - **時間軸（Timeline）**：顯示整段記錄的原始時間序列。
  - **疊圈（Overlay）**：在圈次表勾選想比較的圈，圖表會把各圈疊在同一張圖上比較（顏色代表圈次、線型代表通道，X 軸從 0 開始對齊），方便比較不同圈之間的煞車點、出彎速度等差異。
- **游標同步**：多張時間序列圖表與軌跡地圖之間的游標會互相連動，方便對照同一時間點在地圖上與圖表上的位置（XY 散佈圖的 X/Y 軸是任選的兩個通道，沒有時間/距離意義，因此不參與這個共享游標，也不參與圖表縮放範圍的連動）。
- **觸控手勢**：支援雙指縮放（pinch）與拖曳平移圖表的 X 軸範圍（uPlot 內建的滑鼠拖曳縮放為滑鼠限定，觸控裝置改用手勢達成同樣效果）。
- **疊圈對位微調（Lap Align）**：疊圈比較時，若圈與圈之間有些微時間位移導致彎道/煞車點對不齊，可用「提前（左移）／延後（右移）」個別微調每圈的起點位移，讓特徵點對齊，不影響實際圈時與軌跡資料。可「重置此圈位移」或「全部重置」。
- **地圖對位微調（Map Align）**：不同圈之間若因 GNSS 訊號漂移導致軌跡在地圖上錯開，可用「往北／往南／往東／往西」微調每圈在地圖上的顯示位置對齊路線，同樣不影響圈時與原始數據。可個別重置。
- **圖表框選 → 地圖聚焦**：在時間軸圖表上拖曳框選（或觸控雙指縮放）一段範圍時，若目前**未勾選任何圈次**，軌跡地圖會把該時間範圍對應的路段標示出來並自動縮放聚焦到該段（若有勾選圈次，地圖優先顯示疊圈選取範圍，不會被框選覆蓋）。

#### XY 散佈圖 Scatter chart

XY 散佈圖是一種獨立的圖表類型：X、Y 軸可**任選**任兩個通道畫成散點（不限於力值通道），因此除了下面介紹的「G-G 圖／摩擦圓」，也可以拿來看「轉速 vs. 車速」等任意兩個通道的關係。

- **新增**：點擊「新增 XY 散佈圖」建立一張散佈圖卡片；可建立多張，各自獨立選擇 X/Y 通道。
- **通道選擇**：X 軸、Y 軸各自從全部通道中挑選（可搜尋），兩者都選好才會畫圖。
- **座標軸規則**：當 X、Y 兩個通道**都**是有正負號的力值型資料（數值同時涵蓋正負，例如 `TC_Xforce`/`TC_Yforce`）時，圖表會採用「以 0 為中心的正方形對稱座標軸」——這正是經典的 G-G 圖／摩擦圓畫法，方便直接比較左右、前後方向的抓地力是否對稱。其他通道組合（例如轉速 vs. 車速）則採一般的自動範圍座標軸，不強制置中或正方形。
- **G-G 圖／摩擦圓（featured 範例）**：新增散佈圖時，若記錄含有 `TC_Xforce`/`TC_Yforce`（aRacer Race Module IMU 的橫向/縱向 G 值，單位 milli-g），會預設帶入這兩個通道並自動換算成 g 為單位——這就是經典的「G-G 圖」：畫出車輛在賽道上經歷的橫向 G（過彎）與縱向 G（煞車/加速）的分布，形狀越接近圓形代表輪胎抓地力運用越均衡。若圈次表有勾選圈次，會依圈次分別上色比較；否則顯示整份記錄的分布。
- 若記錄沒有 `TC_Xforce`/`TC_Yforce`，X/Y 軸會留白，需自行挑選想比較的兩個通道。
- **載入提示**：XY 散佈圖所需的繪圖套件（echarts）檔案較大，改為「首次新增散佈圖時才需時載入」，不影響首頁載入速度。因此第一次點擊「新增 XY 散佈圖」時，卡片內會短暫顯示載入提示，之後（同一次瀏覽期間）再新增散佈圖就不會再次等待。

### 4.5 Sector 閘門（彎道驗證）

Sector 功能會在賽道上設下一系列「閘門」，用來檢查每一圈是否真的完整跑過所有彎道（避免切西瓜、抄捷徑等異常圈被誤判為有效圈）。閘門隨時可以自動偵測、手動新增／移除／拖曳調整，沒有「先確認才能用」的中間步驟。

1. 點擊「自動偵測彎道」，系統會分析軌跡並**直接**把偵測到的彎道位置設為目前的 sector 閘門（不需要另外逐一確認）——閘門清單會立刻顯示編號、地圖上立刻畫出，圈次的 sector 計時/驗證也立刻套用新的閘門。
2. 目前的閘門數量顯示在「目前 N 個 sector 閘門」，下方清單列出每個閘門的編號。
3. **新增閘門**：點擊「＋新增閘門」，會在目前地圖游標所在的軌跡位置（若尚未在地圖上移動過游標，則落在參考圈中段）插入一個新閘門，方向會自動對齊該處的行進方向。
4. **移除閘門**：清單中每個閘門項目都有「✕」，點擊即可個別移除。
5. **拖曳調整**：任何閘門都可以直接在軌跡地圖上拖曳兩端調整位置（起終點線的拖曳方式相同）。
6. 新增、移除或拖曳閘門後，系統會依閘門在參考圈上沿行進方向的實際位置**自動重新排序**，確保 sector 順序（sector 計時、驗證都依賴閘門順序）永遠正確，不需要手動排序。
7. 「清除全部」可以一次移除所有閘門，重新設定。
8. **重新自動偵測**：若目前的閘門集合已被手動新增/移除/拖曳調整過，再次點擊「自動偵測彎道」會先跳出確認提示（避免不小心蓋掉手動調整的成果）；若尚未手動調整過（例如剛偵測完、或剛從已儲存賽道還原），會直接套用新的偵測結果。
9. 設好閘門後，圈次表與統計會顯示「N 圈未通過 sector 檢查」——代表這些圈的軌跡沒有依序通過所有已設定的閘門，可能是切彎、少跑一段賽道或訊號異常。這個判定結果也會併入整體的排除圈邏輯中。
10. **理論最佳圈**：只要有至少一個 sector 閘門，Sector 面板就會顯示「理論最佳圈」——把所有未排除圈中每個 sector 的最佳時間加總而成的假想圈速，並列出每個 sector 最佳時間分別來自第幾圈。若目前圈數/資料還不足以算出結果，會顯示提示文字說明無法計算。

> 此功能需要有效的 GPS 軌跡座標才能運作。

### 4.6 軌跡通道標記 Track channel markers

在 Sector 面板下方的「軌跡通道標記」面板，先選擇**一個通道**（車速、RPM、G 力、煞車、油門……任何通道皆可），接著可獨立勾選三個選項，可任意組合：

- **軌跡上色**：地圖上的軌跡依該通道數值上色（即原本的「軌跡上色／熱力圖」，現在整合進同一個面板）。
  - **色階（Colormap）**：可切換 `turbo`（藍→綠→黃→紅）、`viridis`（色盲友善，紫→綠→黃）、`plasma`、`coolwarm` 四種配色。
- **標記最小值**：在地圖上標出該通道的區域最小值點（例如車速通道即為原本的「彎道最低速度標記」）。
- **標記最大值**：在地圖上標出該通道的區域最大值點（例如 RPM 通道的每次拉轉高點）。
- 兩種標記可同時開啟；地圖上以**圓形**標記最小值、**菱形**標記最大值，並依數值高低上色（綠→紅）以便一眼分辨，兩種標記各自獨立編號（最小值 #1、#2……與最大值 #1、#2……互不相關）。
- 面板下方列出每個標記的**種類**（最小/最大）、**圈距**（km）與**數值**清單。
- **標記僅在剛好勾選一圈時才會顯示**：沒有選圈、或同時勾選多圈時，會提示請先在圈次表勾選單一圈；尚未選擇通道時則提示請先選擇通道。上色不受此限制，未勾選圈次時仍會套用在整條軌跡上。
- 原本的「彎道最低速度標記」等同於「選擇速度通道＋只勾選標記最小值」，不再是獨立功能。

### 4.7 直線加速測試 Acceleration test

在軌跡通道標記面板下方，可對**整份記錄**（不限單一圈）搜尋最佳加速區段：

- **測試類型**：
  - **距離**：輸入目標距離（公尺），系統會在全場找出跑完這段距離耗時最短的區間；可額外設定「起始最低速度」，只搜尋起跑速度達到此門檻以上的區段（留空則不限，含靜止起步）。
  - **速度區間**：輸入「起始速度」與「目標速度」（km/h），系統找出從起始速度加速到目標速度耗時最短的區段。
- 找到最佳區段後會顯示：**耗時**、**距離**、**進入 → 離開速度**。
- 「聚焦此區段」：把圖表與地圖縮放聚焦到這個最佳區段所在的位置。
- 需要記錄中含速度通道（`GPS_Speed` 或 `Vehicle_Speed`）；若整份記錄找不到符合條件的區段，會提示無符合結果。

### 4.8 變速齒比計算器 Gear ratio calculator

在直線加速測試面板下方，提供機車傳動系統的齒比計算，分為兩種模式（頁籤切換）：

- **MT（鏈條傳動檔車）**：輸入一次減速比、每檔齒比（1–6 檔，檔數可調整）、終傳齒盤齒數（前／後）、後輪周長 (mm)、紅線／換檔轉速，即可算出：
  - 每檔的**總減速比**與**紅線轉速對應車速**表格（點擊該列可展開該檔的轉速↔車速對照表）。
  - **理論極速**（最高檔在紅線轉速下的車速）。
  - 每次**升檔時的轉速降**（例如 1→2 檔在紅線時會掉多少 RPM）。
- **CVT（速克達）**：輸入 CVT 齒比範圍（低速端／高速端）、終傳減速比、後輪周長、引擎最高轉速，算出在最高轉速下、CVT 從低速端到高速端這段範圍對應的車速（高速端即為極速）。

#### 後輪周長的三種設定方式（MT 模式）

MT 模式的「輪胎規格／後輪周長」區塊提供三種互補的設定方式：

1. **輪胎規格換算**：切到「輪胎規格」分頁，輸入標準機車輪胎規格字串（`寬度/扁平比-輪徑`，例如 `120/70-17`）。支援的格式細節：
   - 分隔符可為 `-`、`x` 或省略，並接受標準構造字母 `R`（子午線胎）、`ZR`、`B`（bias belted）或 `D`（斜紋），大小寫皆可，例如 `120/70ZR17`。
   - 輪徑前可有選填的 `M/C`（機車胎）標記，常見於速克達胎，例如 `130/70 M/C 12`。
   - 尾端可再接選填的載重指數／速度等級（例如 `58W`），例如 `120/70ZR17 58W`。
   - 換算公式概念：胎壁高度 = 寬度 × 扁平比 ÷ 100；輪胎總外徑 = 輪徑(吋) × 25.4 + 2 × 胎壁高度；周長 = π × 總外徑。
   - 解析成功會顯示「解析後周長：___ mm」；格式無法辨識時會提示「無法解析輪胎規格」。
   - 點擊**「套用為周長（可再微調）」**，會把換算出的周長寫入「直接輸入」模式的周長欄位並自動切到該模式——因為同規格的實體輪胎仍有磨損差異（有尖有圓），套用後的數值只是起點，仍可手動微調。切回「直接輸入」模式後，若曾輸入過規格字串，欄位旁會顯示「規格 ___ 換算約 ___ mm（實胎有尖有圓，可微調）」作為參考值，方便對照目前微調過的周長與規格原始估計值差多少。
2. **直接輸入周長**：切到「直接輸入」分頁，直接填入後輪實測或微調後的周長 (mm)。
3. **從記錄倒算**：見下方說明。

#### 從記錄倒算有效周長

在周長設定區塊下方，有一顆**「從記錄倒算周長（speed / RPM）」**按鈕，利用記錄中的速度／轉速比值反推出實際有效的後輪周長（不需要事先知道周長，只需要齒比與終傳）：

- **前提**：
  - 已載入記錄，且記錄含轉速（`RPM`）與速度（`GPS_Speed` / `Vehicle_Speed`）頻道。
  - 已輸入有效的每檔齒比與終傳（無論用比值或齒數皆可）——這兩項用來把「速度/轉速」商值換算回周長，本身不需要先填周長。
  - 目前為 MT 模式（CVT 模式不支援此功能，因為 CVT 傳動比連續變化、無法用「檔位平台」分群反推）。
  - 不滿足以上任一前提時，按鈕會顯示為停用，旁邊會顯示對應原因提示（例如「請先輸入有效的齒比與終傳（不需周長）才能從記錄倒算」）。
- **結果怎麼解讀**：點擊後若成功，會顯示「倒算周長：___ mm（___ 筆樣本），已填入周長欄位，可再微調。」——mm 為推算出的有效周長中位數，樣本數代表有多少筆記錄資料通過品質篩選（濾除起步/離合器打滑/換檔瞬態等雜訊後）用於這次估計；結果會自動填入「直接輸入」周長欄位並切換到該模式，同樣可以再手動微調。
- **什麼情況會失敗**：若資料不足（合格樣本太少）、或整段記錄從頭到尾只騎在同一個檔位（此時無法判定各檔位對應關係，除非已先填入一個概略周長作為參考值破除歧義）、或齒比/終傳無效，會顯示「無法穩定倒算：穩定樣本不足，或整段記錄僅單一檔位無法判定檔位（可先填入概略周長作為參考再試一次）。」——此時可以先用輪胎規格換算或手動填一個大概的周長作參考值，再重新點擊倒算按鈕。

**由記錄反推齒比**（僅在已載入記錄、且記錄含轉速（`RPM`）與速度（`GPS_Speed` / `Vehicle_Speed`）頻道時顯示）：

- 輸入後輪周長，系統會用 `ratio(t) = 引擎轉速 / 後輪轉速`（後輪轉速由車速換算）計算整份記錄逐點的實際傳動比，並自動濾掉車速低於 5 km/h（起步、離合器打滑）或轉速/速度非數值的樣本，避免雜訊。
- **MT 模式**：把濾過的比值做叢集分析（分群容忍度 3%），抓出最穩定的幾個「平台」— 也就是每一檔實際的總減速比，並與上方計算機輸入的設定值並列比較（偵測值 vs. 設定值），方便核對實際齒比是否因輪胎更換、鏈條磨耗等因素與帳面規格有落差。
- **CVT 模式**：直接把「車速 vs. 傳動比」畫成散點圖（沿用既有的 UPlotChart 元件），呈現 CVT 隨車速連續變化的齒比曲線。

以實際 .loga 記錄（含起步加速的測試片段）驗證：在給定檔數提示（如 6 檔）時，偵測到的 6 個平台彼此間隔約 5–13%，與真實機車檔位間距吻合；若不給檔數提示，連續加速斜坡會被切成過多細碎平台（因為斜坡本身近似連續變化），所以檔數提示（或加大容忍度）對加速測試類型的記錄很重要。

### 4.9 賽道本機持久化與軌跡檔 Track file

分析功能會依 GPS 位置辨識「這是哪個賽道」，自動把該賽道的起終點線、sector 閘門、圈次表欄位設定儲存在瀏覽器本機（IndexedDB）。下次載入同一個賽道（同樣的 GPS 位置）的記錄時，這些設定會**自動還原**，不需要重新拖曳起終點線或重新偵測彎道。

「賽道設定」是分析頁其中一張卡片（見 4.10 節「版面配置」——卡片位置可自由拖曳，故不再有固定順序），提供：

- **匯出賽道設定**：把目前賽道的起終點線、sector 閘門、圈次表欄位打包匯出成一份 JSON「軌跡檔」，方便備份或分享給別人。
- **匯入賽道設定**：匯入一份先前匯出的 JSON 軌跡檔；若匯入的賽道正是目前開啟的賽道，會立即套用。
- **已儲存的賽道設定清單**：以「已儲存 N 個賽道設定」展開查看，列出每個已儲存賽道的名稱與更新時間，可個別「刪除」。
- 若目前記錄沒有有效 GPS 座標，無法定位賽道，匯出功能會停用並顯示提示。

> 變速齒比計算器的輸入設定（MT/CVT 選擇、各檔齒比、輪周長等）會自動保存在瀏覽器（localStorage），重新整理頁面或關閉分頁後仍會保留，不需要每次重新輸入；此設定屬於「車輛規格」性質，與上述賽道設定（IndexedDB，依賽道 GPS 位置區分）是各自獨立的儲存機制。

#### 公開賽道庫自動套用

除了上述「自己記錄過就會記住」的本機設定，本站也內建一份**公開賽道庫**（收錄各賽道官方或社群貢獻的起終點線、sector 閘門）。載入記錄時，若這是你在本機**第一次**看到的賽道（本機還沒有對應的賽道設定），系統會用記錄的 GPS 座標去比對賽道庫（誤差容許約 100 公尺）：

- **命中一筆**：自動套用該賽道庫配置的起終點線與 sector 閘門，畫面上方會出現提示訊息：「已自動套用賽道庫的「〈賽道名稱〉」設定，可在下方調整，或改回本機設定不再跟隨賽道庫更新。」
  - 之後若賽道庫更新了這筆配置（例如社群修正了閘門位置），只要你**沒有**手動調整過，下次載入同賽道記錄仍會拿到最新版本。
  - 若不想繼續跟隨賽道庫、想改成自己手動調整（或維持匯入時的版本），點提示訊息旁的「改回本機設定」即可切成純本機設定，之後賽道庫更新就不會再覆蓋。
- **命中多筆**（同一地點有多種賽道配置，例如同場地的順向／逆向或不同賽道長度）：不會自動選一個，而是跳出「偵測到多個可能的賽道配置」清單，列出每個候選配置的名稱，點「套用此配置」選擇符合這次記錄的那一筆；若都不符合，可點「都不是（略過）」不套用任何配置。
- **完全沒命中**：不受影響，維持原本「自行拖曳起終點線／自動偵測彎道」的既有流程；下次在本機記住這份手動設定後，就不會再去查賽道庫。
- 你自己在本機建立、匯入、或手動調整過的賽道設定一律優先，賽道庫只補「本機還沒有任何紀錄」的情況。

#### 貢獻賽道到公開庫

「賽道設定」卡片下方有一個可展開的「貢獻這條賽道到公開賽道庫」區塊，讓你把自己校正好的起終點線、sector 閘門包裝成一份**可直接送出 Pull Request** 的 JSON 檔：

- 需填寫：**賽道 ID**（例：`tw-example-track`）、**顯示名稱**、**國碼**（ISO 3166-1 alpha-2，例如 `TW`）、**授權**（預設 `CC0-1.0`，可自行修改）。
- 填完並點「匯出賽道庫 JSON」即可下載檔案；若目前記錄沒有畫出起終點線，會提示「請先畫出起終點線，目前沒有可匯出的內容」而無法匯出。
- 此功能僅整理與匯出 JSON 檔案本身，不會自動幫你開 PR、也不會把資料上傳到任何伺服器——後續要不要分享、如何送出，由你自己決定。

### 4.10 版面配置 Layout（電腦版拖曳/縮放格線）

在較寬的螢幕（電腦版）上，分析頁的每個功能區塊（軌跡地圖、圈次表、sector 閘門、軌跡通道標記、加速測試、變速齒比計算器、賽道設定、地圖對位微調、疊圈對位微調，以及每一張圖表）都是一張可獨立**拖曳排列**、**拖曳縮放**的卡片，讓你依自己的分析習慣把常看的卡片並排放大、不常用的縮小或移到旁邊，善用寬螢幕左右兩側的空間。

- **拖曳移動**：按住卡片**標題列**（卡片最上方那一條，含卡片名稱）拖曳即可搬移位置；卡片內容區（例如地圖的縮放平移、表格的點選）不會觸發拖曳。
- **拖曳縮放**：把滑鼠移到卡片**右下角**，出現縮放游標後拖曳即可調整卡片大小；地圖與圖表會即時重新繪製以配合新的大小，不會模糊或跑版。每張卡片都有最小尺寸限制，縮放時不會小於能看清內容的下限。
- **自動保存**：版面配置（每張卡片的位置與大小）會自動保存在瀏覽器（localStorage），重新整理頁面或關閉分頁後仍會保留。
- **重設布局**：工具列的「重設布局」按鈕可一鍵恢復預設的版面配置（地圖與圈次表在左欄、圖表與工具面板在右欄）。「＋新增圖表」「＋新增 XY 散佈圖」與「重設布局」同屬工具列上同一個按鈕群組（緊鄰顯示），新增圖表卡片與重設整體布局是相關的兩個操作，放在一起方便切換。
- **新增圖表卡片**：透過工具列的「＋新增圖表」／「＋新增 XY 散佈圖」新增的圖表會自動加入版面配置一個預設位置；移除圖表時，該卡片也會一併從版面配置移除。
- **收合卡片**：每張卡片標題列右側都有一個收合／展開按鈕（v 形箭頭），點擊可只保留標題列、隱藏卡片內容，方便暫時把不需要的卡片縮小讓畫面更清爽；此功能電腦版、手機版皆可使用，狀態會自動保存。

#### 手機版單欄 + 收合 + 釘選

螢幕寬度低於約 768px 時，版面會自動收合成單欄、依電腦版布局換算出的邏輯順序排列，且**停用拖曳與縮放**（單欄模式下排列順序固定，拖曳/縮放沒有意義）。手機版額外提供：

- **收合卡片**：同電腦版，點擊標題列的 v 形箭頭可收合／展開任一張卡片內容，方便在單欄中快速略過不需要的區塊。
- **釘選卡片（📌）**：每張卡片標題列在手機版會多一個釘選圖示按鈕。點擊後，該卡片會**固定在畫面頂端**（sticky），其餘卡片繼續在下方正常捲動——例如把「軌跡地圖」釘選起來，捲動查看下方的 XY 散佈圖或圈次表時，地圖仍留在畫面上方，方便同時比對。
  - 同一時間**最多只能釘選一張卡片**：釘選另一張卡片會自動取消原本釘選的卡片。
  - 釘選的卡片高度上限約螢幕高度的 45%（`max-height: 45vh`），避免佔滿整個畫面導致捲動不到其他內容。
  - 再次點擊釘選圖示即可取消釘選。
  - 收合與釘選狀態會分別保存在瀏覽器（localStorage），重新整理頁面或關閉分頁後仍會保留。

### 4.11 GPS 場次合併 Session merge

有些 ECU（例如未安裝 Race Module）記錄不到 GPS，只有轉速、車速等感測器資料；若同一趟騎乘另外用手機或 RaceChrono 裝置錄了一份含 GPS 的記錄，「GPS 場次合併」卡片可以把兩者對齊時間軸後合併成一份新記錄，補上 GPS 座標、車速、方向等頻道，讓原本不能定位賽道、切圈的記錄也能使用軌跡地圖、圈次表等功能。

此功能需要**先在分析頁載入至少兩份記錄**（例如缺 GPS 的 `.loga` + 有 GPS 的 `.nmea`），操作方式：

1. **主要記錄（欲補上 GPS）**：選擇要保留其原始資料、缺少 GPS 的那一份記錄。
2. **GPS 來源記錄**：選擇含乾淨 GPS 資料的另一份記錄。
   - 兩個下拉選單列出目前已載入的所有記錄；沒有可用速度頻道的記錄後面會標示「（無可用速度頻道）」，因為自動對齊需要靠速度訊號比對。
3. **自動對齊**：點擊後系統會用兩份記錄的速度資料做交叉相關運算，找出兩者時鐘的時間偏移量，並顯示：
   - **時間偏移**（ms）。
   - **相關性分數**：對齊結果的信心程度參考值。
   - 若所選記錄缺少可用的速度或時間頻道，或兩份記錄的時間範圍/速度訊號重疊不足，會顯示對應的錯誤提示，無法繼續。
4. **微調偏移量**：自動對齊的結果不一定完美，可用時間偏移旁的「−100ms」「+100ms」按鈕以 100ms 為單位手動微調，不需要重新跑一次自動對齊。
5. **合併並加入記錄**：確認偏移量後點擊，系統會把 GPS 座標/車速/方向等頻道依偏移量接上主要記錄的時間軸，產生一份新記錄（檔名為「原檔名`_merged.loga`」），並提示「已合併為新記錄「〈檔名〉」，可在上方記錄選單選取查看」。合併後的新記錄會出現在上方的記錄選單中，可像其他記錄一樣切換檢視、使用分析功能，或在轉換頁匯出。

## 5. 設定 Settings

### 5.1 主題與語言

- **主題**：自動（跟隨系統）、淺色、深色，三選一。
- **語言**：自動（依瀏覽器語系偵測）或手動切換繁體中文 / English。

### 5.2 時區

分析與轉換頁籤顯示的時間會依此處設定的時區換算：可選「自動（瀏覽器）」，或從 UTC-12 到 UTC+14 的整點時區清單中手動選擇。

> 避震校正與另存修改後的 `.loga` 已移至轉換頁（見 3.4、3.6 節）；設定頁僅保留主題、語言、時區等一般應用設定。

## 6. 疑難排解

### 6.1 診斷模式 `?debug=1`

若在手機上遇到「載入檔案後整個頁面自己重新整理」等異常狀況，可以在網址後面加上 `?debug=1`（例如 `https://track-log-studio.timliudev.workers.dev/?debug=1`）開啟**裝置端診斷面板**：

- 會記錄頁面生命週期事件（例如分頁被系統回收 `wasDiscarded`、JS 錯誤、記憶體用量趨勢等），並在頁面重新整理「之後」直接顯示在畫面上，方便在沒有電腦連線除錯工具（DevTools）的情況下，回報手機端發生的問題。
- 一旦以 `?debug=1` 開啟過，此模式會記住（存在瀏覽器本機儲存），之後不用每次都加參數；要關閉請改用 `?debug=0`。
- 一般使用者平時不需要開啟此功能。

### 6.2 手機使用注意事項

- 大型記錄檔在手機瀏覽器上解析／繪圖較耗記憶體，若遇到頁面異常重整，可嘗試安裝為 PWA、關閉其他分頁釋放記憶體，或使用上述 `?debug=1` 協助回報問題。
- 觸控操作：軌跡地圖與圖表皆支援雙指縮放、拖曳平移；起終點線與 sector 閘門的拖曳調整在觸控裝置上同樣可用。

### 6.3 版本資訊

頁面最下方頁尾會顯示「build 版本碼 · 日期」字樣，回報問題時請附上這串資訊，方便對應到確切的程式版本。
